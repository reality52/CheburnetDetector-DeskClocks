# CheburnetDetector-DeskClocks

## Детектор Чебурнета — настольные часы с мониторингом сети

Сетевой монитор-часы для ESP8266-01 с полноценным веб-интерфейсом для настройки.

Устройство для мониторинга доступности сети, интернет-сервисов и whitelist-сайтов.
Оснащено LCD‑дисплеем (I²C), динамиком и веб‑сервером с настройками, историей проверок и REST API.

## Возможности

- ✅ Проверка доступности локального шлюза, whitelist и внешних сайтов (ping + HTTP).
- 📶 **WiFi AP-режим с captive portal**: автоматическая точка доступа `NetworkMonitor` при первом запуске или потере соединения (30 сек). Мобильные устройства автоматически открывают страницу настройки. Сканирование сетей, выбор из списка, сохранение пароля — без перепрошивки.
- 🖥️ LCD‑дисплей 16×2: текущее время и дата (NTP-синхронизация), мигающее двоеточие.
- 🔔 Звуковая индикация при смене статуса сети — с настраиваемыми «тихими часами».
- 🌐 Веб‑сервер с:
  - страницей мониторинга (время, статус, история, кнопки управления),
  - **страницей настроек** (`/settings`) для конфигурации всех параметров без перепрошивки,
  - REST API (`/api/config`) для программного чтения и сохранения настроек.
- 📝 Управление хостами через WebUI — добавление, редактирование, удаление и перемещение.
- 💾 Все настройки сохраняются в LittleFS и загружаются при старте.
- 🔄 Настраиваемый интервал проверки (по умолчанию 3 минуты).
- 🌙 Тихий режим: динамик не работает в заданные часы.
- 🎵 Мелодия Tetris по кнопке из веб-интерфейса.

  │  │ GND ─────────────── GND                          │    ││  │
  │  │ SDA ─────────────── GPIO2 ───────────────────────┘    ││  │
  │  │ SCL ─────────────── GPIO0 ────────────────────────────┘│  │

  │  ┌──────────────────────────────┐                            │
  │  │  + (анод) ─── GPIO1 (TX)     │                            │

> * LCD может работать от 3.3V, но некоторые модули PCF8574 требуют 5V.
>   В таком случае питайте LCD отдельно от 5V (GND общий), а линии I²C через
>   делитель напряжения или Level Shifter.

### Таблица подключения

| Компонент        | Пин ESP8266-01        | Примечание                                |
|------------------|-----------------------|-------------------------------------------|
| LCD (I²C)        | SDA → GPIO2           | Адрес 0x27                                |
|                  | SCL → GPIO0           |                                           |
|                  | VCC → 5V / 3.3V       | **См. примечание выше про питание**        |
|                  | GND → GND             |                                           |
| Динамик (пьезо)  | + → GPIO1 (TX)        | Полярность не критична, но так громче     |
|                  | − → GND               |                                           |
| Питание ESP-01   | VCC → 3.3V            | От USB-UART адаптера                      |
|                  | GND → GND             | Общий с LCD и динамиком                   |
|                  | EN → 3.3V             | Подтяжка к питанию (обязательно!)         |
|                  | RST → 3.3V            | Подтяжка через 10кΩ (рекомендуется)       |

> **Важно:** для прошивки ESP-01 используйте USB-UART адаптер с напряжением 3.3V.
> LCD‑модуль PCF8574 обычно имеет встроенные подтягивающие резисторы на линиях I²C.

## Сборка и прошивка

1. Установите в Arduino IDE платформу **ESP8266** (через менеджер плат).
2. Установите библиотеки:
   - `LiquidCrystal_PCF8574` (by Matthias Hertel)
   - `pitches.h` (HiBit buzzer library)
3. Подключите ESP-01 через USB-UART адаптер.
4. Выберите плату: **Generic ESP8266 Module**.
5. Загрузите скетч.
6. При первом запуске будет создана файловая система LittleFS — дождитесь инициализации.

### Настройка Wi-Fi

**Способ 1 — через AP-точку доступа (рекомендуется):**

1. Прошейте скетч с любыми (или дефолтными) настройками Wi-Fi.
2. При первом запуске (или если Wi-Fi недоступен) устройство создаст точку доступа **NetworkMonitor** (пароль: `12345678`).
3. Подключитесь к этой сети с телефона/ноутбука.
4. Откройте `192.168.4.1` в браузере (у большинства телефонов страница настройки откроется автоматически — captive portal).
5. Нажмите **🔍 Scan**, выберите свою сеть из списка, введите пароль и нажмите **💾 Сохранить и перезагрузить**.
6. Устройство перезагрузится и подключится к вашей сети.

**Способ 2 — задать в коде перед прошивкой:**

```cpp
char wifiSsid[33] = "your-ssid";
char wifiPass[65] = "your-pass";
```

> Если сохранённая сеть станет недоступна (или пропадёт на 30 секунд во время работы), устройство автоматически перейдёт в AP-режим и подаст звуковой сигнал. Все остальные настройки (хосты, NTP, тихие часы) при этом сохраняются.
>
> **Сменить WiFi можно и в обычном режиме** — откройте `/setup` в браузере (или нажмите кнопку **WiFi Setup** на главной странице).

## Веб-интерфейс

Откройте в браузере IP-адрес ESP8266 (узнать можно из Serial Monitor или через роутер).

### Главная страница (`/`)

- **Server time** — текущее время с NTP.
- **Current status** — OK / Network Problem / Dead Internet / Whitelisted / Anomaly.
- **Ход теста** — отображает проверяемый хост и результат последней проверки в реальном времени.
- **Кнопки:**
  - **Run test now** — запускает внеочередную проверку сети.
  - **Clear Logs** — очищает историю и счётчики.
  - **♫ Tetris** — проигрывает мелодию через динамик.
  - **WiFi Setup** — переход на страницу настройки WiFi (`/setup`).
- **Таблица History** — дата, время и статус проблемных проверок (до 100 записей, только события «не OK»).
- Все действия работают без перезагрузки страницы (AJAX).

### Страница настроек (`/settings`)

Полная конфигурация устройства через браузер — **без перепрошивки**:

| Раздел                | Параметры                                                        |
|-----------------------|------------------------------------------------------------------|
| **WiFi**              | SSID, пароль (применяется после перезагрузки)                    |
| **NTP Time Sync**     | NTP-сервер, GMT-смещение, DST-смещение                           |
| **Silent Hours**      | Часы тишины (старт / конец, 0–23)                                |
| **Network Check**     | Интервал проверки в секундах (мин. 10)                           |
| **Host Lists**        | Управление хостами в трёх категориях (см. ниже)                  |

Настройки сохраняются в файл `/config.txt` на LittleFS и загружаются автоматически при каждой перезагрузке.

### Страница настройки WiFi (`/setup`)

Доступна как в AP-режиме, так и в обычном (подключённом) режиме:

- 🔍 **Сканирование** — поиск доступных WiFi-сетей с отображением уровня сигнала (RSSI) и типа шифрования.
- 📋 **Выбор из списка** — SSID подставляется автоматически при выборе сети.
- 💾 **Сохранение** — новые credentials сохраняются в `/wifi.conf` на LittleFS, устройство перезагружается.
- 🔄 **Captive portal** — в AP-режиме мобильные устройства автоматически открывают эту страницу.

## Управление хостами

На странице `/settings` можно полностью управлять тремя категориями хостов:

| Категория           | Описание                                        | Примеры                                |
|---------------------|-------------------------------------------------|----------------------------------------|
| 📡 **Local**        | Локальный шлюз (проверяется только ping)         | `192.168.0.1`                          |
| 🚨 **Whitelist**    | Whitelist-сайты (проверяются ping + HTTP)        | `vk.com`, `ya.ru`, `max.ru`           |
| 🌍 **World**        | Внешние мировые сайты (проверяются ping + HTTP)  | `google.com`, `cloudflare.com`         |

Для каждого хоста доступны действия:

- ✏️ **Редактирование** — inline-редактирование имени с сохранением по Enter или ✓
- ✕ **Удаление** — с подтверждением и анимацией
- ▲▼ **Перемещение** — изменение порядка проверки
- ➕ **Добавление** — поле ввода внизу каждой категории (поддержка Enter)

Каждая категория показывает счётчик (например, `3/10 hosts`). Максимум 10 хостов на категорию.

> **Важно:** перед сохранением настроек все открытые редактирования автоматически сохраняются.

## API веб-сервера

| Эндпоинт          | Метод | Описание                                           |
|-------------------|-------|----------------------------------------------------|
| `/`               | GET   | Главная страница мониторинга                        |
| `/settings`       | GET   | Страница настроек                                   |
| `/api/config`     | GET   | JSON: текущая конфигурация                          |
| `/api/config`     | POST  | Сохранение конфигурации (JSON-тело)                 |
| `/api/scan`       | GET   | JSON: список доступных WiFi-сетей                   |
| `/restart`        | GET   | Перезагрузка устройства                             |
| `/status`         | GET   | JSON: текущий статус и uptime                       |
| `/clear`          | GET   | Очистка истории и счётчиков                         |
| `/refresh`        | GET   | Запуск внеочередной проверки                        |
| `/history`        | GET   | JSON-массив истории проверок                        |
| `/now`            | GET   | JSON: текущее серверное время                       |
| `/testprogress`   | GET   | JSON: ход текущего теста                            |
| `/setup`          | GET   | Страница настройки WiFi (доступна в обоих режимах)    |
| `/scan`           | GET   | JSON: список доступных WiFi-сетей (сканирование)      |
| `/savewifi`       | POST  | Сохранение WiFi-настроек и перезагрузка (JSON-тело)   |
| `/tetris`         | GET   | Проигрывание мелодии Tetris                         |

### Пример: чтение конфигурации

```bash
curl http://<ESP_IP>/api/config
```

Ответ:
```json
{
  "wifi_ssid": "MyNetwork",
  "ntp_server": "0.ru.pool.ntp.org",
  "gmt_offset": 10800,
  "dst_offset": 0,
  "silent_start": 21,
  "silent_end": 8,
  "check_interval": 180,
  "local_hosts": ["192.168.0.1"],
  "whitelist_hosts": ["vk.com", "ya.ru", "max.ru"],
  "world_hosts": ["google.com", "cloudflare.com"]
}
```

### Пример: обновление хостов

```bash
curl -X POST http://<ESP_IP>/api/config \
  -H "Content-Type: application/json" \
  -d '{"world_hosts":["google.com","example.com"]}'
```

## Логика определения статуса

| Условие                                                   | Статус            | LCD-сообщение        |
|-----------------------------------------------------------|-------------------|----------------------|
| Нет доступа к локальному шлюзу                            | Network Problem   | Network Problem!     |
| Локальный есть, но нет whitelist и нет внешних            | Dead Internet     | DEAD INTERNET        |
| Локальный есть, whitelist работает, внешние нет           | Whitelisted       | ! WHITELISTED !      |
| Частичные потери (не все сайты)                           | Anomaly           | Anomaly !            |
| Всё доступно                                              | OK                | часы и дата          |

## Работа LCD‑дисплея

- **Нормальный режим** — часы и дата (двоеточие мигает).
- **Во время теста** — анимация в четырёх углах экрана (не затирает время).
- **При проблемах** — сообщение о статусе и мигание подсветки.

## Звуковые сигналы

- Короткий писк — при включении.
- Три восходящих тона — успешное подключение к Wi-Fi.
- Два низких гудка — вход в AP-режим (точка доступа).
- Длинные гудки — ошибка Wi-Fi.
- Мелодия статуса — при смене статуса на не-OK (не более 10 секунд, игнорируется в тихие часы).
- **Tetris Melody** — по кнопке из веб-интерфейса.

## WiFi AP-режим

| Сценарий                          | Поведение                                           |
|-----------------------------------|-----------------------------------------------------|
| Первый запуск (SSID = "your-ssid")| Автоматический AP-режим для настройки               |
| Сохранённая сеть недоступна       | Автоматический AP-режим (с сигналом), настройки сохраняются |
| Сеть пропала во время работы      | Через 30 секунд — автоматический AP-режим          |
| Сеть восстановлена                | Обычный режим при следующей перезагрузке            |
| Переход на `/setup` в обычном режиме | Смена WiFi без потери текущего подключения (до перезагрузки) |

**Точка доступа:** `NetworkMonitor` / пароль `12345678` / IP `192.168.4.1`

## Известные ограничения

- HTTP‑проверка работает только через порт 80 (без SSL). HTTPS-сайты не проверяются, но редирект с HTTP на HTTPS считается успехом.
- При изменении Wi-Fi через веб-интерфейс устройство перезагружается автоматически.
- LittleFS создаётся автоматически при первом запуске.
- Максимум 10 хостов на категорию.
- AP-пароль захардкоден (`12345678`).

## Возможные проблемы и решения

| Проблема                                      | Решение                                                               |
|-----------------------------------------------|-----------------------------------------------------------------------|
| HTTP всегда FAIL                              | Проверьте, что сайт отвечает на порт 80. Редирект на HTTPS поддерживается. |
| Неправильное время в таблице истории          | Убедитесь, что NTP-синхронизация прошла (Serial: `Time synced`).      |
| Настройки не сохраняются                      | Проверьте, что LittleFS смонтирован (Serial: `Config saved`).         |
| Устройство не подключается к Wi-Fi после смены SSID | Устройство автоматически вернётся в AP-режим. Подключитесь к `NetworkMonitor` и настройте заново. |
| Не появляется точка доступа NetworkMonitor     | Убедитесь, что SSID в конфиге не "your-ssid". Удалите `/config.txt` через Serial или перепрошьюте. |
